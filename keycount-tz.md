# KeyCount — Техническое задание v1.0

**Приложение:** KeyCount  
**Тип:** macOS Menu Bar App  
**Лицензия:** Open Source (MIT)  
**Минимальная ОС:** macOS 26.0  
**Статус документа:** Финальная версия

---

## 1. Цель продукта

Лёгкое нативное приложение для macOS, которое живёт в строке меню, в фоне перехватывает все нажатия клавиш и показывает накопленную статистику символов. Главные принципы: **нулевой ощутимый импакт на производительность**, полная приватность (никакого текста не сохраняется) и возможность собрать из исходников без App Store.

## 2. Функциональные требования

### 2.1 Статистика

Отображается в popover при клике на иконку:

| Метрика | Описание |
|---|---|
| **Сегодня** | Сумма символов с 00:00 текущего дня (локальный часовой пояс) |
| **Последний час** | Скользящее окно — ровно 60 минут от текущего момента |
| **Среднее в час** | `Сегодня ÷ количество прошедших часов дня` (минимум 1 ч в знаменателе) |
| **Среднее в сутки** | `Всего ÷ количество активных дней` (дней с count > 0) с момента установки или последнего полного сброса |
| **Всего** | Накопленный счётчик с первого запуска или последнего полного сброса |

### 2.2 Подсчёт символов

**Режим по умолчанию: Smart** — устанавливается при первом запуске.

Переключается в Настройках:

| Режим | Название в UI | Что считается |
|---|---|---|
| **Smart** (дефолт) | «Умный подсчёт» | Печатные символы + пробел + Enter (настраивается) + Backspace −1 |
| **AllExceptModifiers** | «Все нажатия» | Всё кроме Cmd, Shift, Option, Ctrl, Fn, Caps Lock |
| **AllKeyDown** | «Полный подсчёт» | Абсолютно все события `keyDown` |

**Детали режима Smart:**

- Backspace **вычитает 1** из счётчика везде (сегодня, последний час, всего). Не уходит ниже 0 в рамках каждого бакета.
- Пробел — **считается**.
- Enter — **настраивается отдельным тогглом** «Считать Enter символом» (по умолчанию: **выкл.**).

### 2.3 Сброс

Два отдельных действия, оба требуют подтверждения через системный диалог:

**Сбросить сегодня:**
- Удаляются только бакеты текущего дня (с 00:00 до сейчас).
- «Всего» уменьшается на количество сброшенных символов.
- История прошлых дней не затрагивается.

**Полный сброс:**
- Удаляется вся таблица бакетов.
- `total_count` → 0.
- `install_ts` не меняется, `last_reset_ts` = now.
- «Среднее в сутки» начинает считаться заново.

Текст диалогов:

```
— Сбросить сегодня —
«Данные за сегодняшний день будут удалены. Статистика прошлых дней сохранится.»
Кнопки: [Сбросить сегодня]  [Отмена]

— Полный сброс —
«Все данные будут удалены: история, общий счётчик, статистика по дням.
Это действие необратимо.»
Кнопки: [Сбросить всё]  [Отмена]
```

### 2.4 История по дням

В **v1 не отображается** в UI, но вся инфраструктура для неё проектируется сразу.

Заложить в v1:
- Поминутные бакеты — достаточны для вычисления любой агрегации по истории.
- Слой `StatsRepository` содержит методы: `sumForDay(date:)`, `sumForRange(from:to:)`, `allDays() → [(Date, Int)]`.
- Метод `allDays()` возвращает только дни с `count > 0`, пустые дни не хранятся.

## 3. UI / UX

### 3.1 Иконка в строке меню

```
[⌨]  12K
```

- **Иконка:** SF Symbol `keyboard`, толщина `regular`.
- **Текст:** счётчик «сегодня» в формате, описанном ниже. Показ текста — опция в Настройках (по умолчанию: **вкл.**).
- Шрифт текста: системный, атрибут `.monospacedDigit` — цифры не «прыгают» при обновлении.
- Иконка адаптируется к светлой / тёмной строке меню автоматически.

**Формат числа в строке меню:**

| Диапазон | Формат | Примеры |
|---|---|---|
| 0 – 999 | `0.XK` | `0.1K`, `0.9K` |
| 1 000 – 9 999 | `X.XK` | `1.2K`, `9.9K` |
| 10 000 – 98 999 | `XXK` | `12K`, `89K` |
| 99 000 и выше | `99K` (жёсткий кэп) | `99K` |

### 3.2 Popover

Открывается по клику на иконку. Фиксированная ширина **260 pt**, высота подстраивается под контент. Полностью нативный системный стиль SwiftUI.

```
┌────────────────────────────────────────┐
│  ⌨  KeyCount                    ···   │  ← заголовок + кнопка настроек
├────────────────────────────────────────┤
│  Сегодня               12 847          │
│  Последний час            543          │
├╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┤
│  Среднее в час           1 024         │
│  Среднее в сутки        18 300         │
├╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┤

├────────────────────────────────────────┤
│  [ Сбросить сегодня ] [ Полный сброс ] │
└────────────────────────────────────────┘
```

- Все числа: правое выравнивание, `.monospacedDigit`, форматирование с разделителями тысяч.
- Данные обновляются **при открытии** popover и каждые **5 секунд** пока он открыт.
- «Последний час» обновляется по скользящему окну — не сбрасывается при наступлении 00:00.
- Кнопка `···` открывает окно Настроек.

### 3.3 Настройки

SwiftUI `Settings` scene, одна панель, системный стиль:

- **Показывать счётчик в меню-баре** `Toggle`
- **Запускать при входе в систему** `Toggle`
- — разделитель —
- **Режим подсчёта** `Picker`: Smart / Все нажатия / Полный подсчёт
- **Считать Enter символом** `Toggle` (активно только в режиме Smart)
- — разделитель —
- Версия приложения (мелкий текст, внизу)

### 3.4 Onboarding

Показывается при первом запуске, если `CGPreflightListenEventAccess()` возвращает `false`:

```
Заголовок: «Нужен доступ к вводу»
Текст: «KeyCount считает нажатия клавиш, но не записывает текст.
        Откройте Системные настройки и разрешите доступ.»
Кнопка: [Открыть Системные настройки]
```

При отзыве разрешения пользователем — иконка в меню-баре показывает `⚠️`, подсчёт приостанавливается. При повторной выдаче разрешения — подсчёт возобновляется автоматически.

## 4. Техническая архитектура

### 4.1 Перехват клавиш

```
Механизм:   CGEventTap (CoreGraphics)
Тип:        kCGHeadInsertEventTap
Место:      kCGSessionEventTap
Маска:      kCGEventKeyDown
Режим:      passive / listenOnly — не модифицирует и не блокирует события
```

- Tap работает на выделенном фоновом `RunLoop` / `DispatchQueue` с QoS `.utility`.
- Единственное действие в коллбэке: атомарный инкремент / декремент in-memory счётчика + запись временно́й метки в кольцевой буфер.
- Никакой записи кейкодов, символов или текста — только числа.

### 4.2 Хранение данных

**Файл:** `keycount.db` (SQLite) в `~/Library/Application Support/KeyCount/`

```sql
-- Основная таблица: поминутные бакеты
CREATE TABLE keystrokes_buckets (
    bucket_ts  INTEGER PRIMARY KEY,  -- unix timestamp, округлён до минуты
    count      INTEGER NOT NULL      -- нажатий в эту минуту (может быть отрицательным из-за Backspace, но суммарно не < 0)
);

-- Метаданные
CREATE TABLE meta (
    key    TEXT PRIMARY KEY,
    value  TEXT
);
-- Ключи: install_ts, total_count, last_reset_ts, counting_mode, count_enter, show_count_in_menubar, launch_at_login
```

**Почему поминутные бакеты:**
- «Последний час» = `SUM(count) WHERE bucket_ts >= now - 3600` — один быстрый SQL-запрос.
- «Сегодня» = `SUM(count) WHERE bucket_ts >= midnight_today`.
- История по дням не требует миграции схемы.
- Размер: ~4 МБ за год активного использования.

**Запись в БД:**
- Батчами каждые **30 секунд** — не на каждое нажатие.
- Происходит в отдельной фоновой очереди с QoS `.background`.
- Между сбросами счётчик живёт в `AtomicInt` в оперативной памяти.
- При завершении приложения — принудительный flush остатка в БД.

### 4.3 Архитектурные слои

```
KeyCountApp (SwiftUI App)
│
├── MenuBarManager
│   ├── NSStatusItem + popover lifecycle
│   └── Форматирование строки меню (формат XK)
│
├── KeystrokeMonitor
│   ├── CGEventTap + выделенный RunLoop
│   ├── AtomicInt (inMemoryCounter)
│   └── Publisher: keystrokeCountChanged
│
├── StatsAggregator
│   ├── Подписан на KeystrokeMonitor
│   ├── Flush в БД каждые 30 с
│   └── computeStats() → StatsSnapshot
│
├── StatsRepository
│   ├── sumForDay(date:) → Int
│   ├── sumForRange(from:to:) → Int
│   ├── allDays() → [(Date, Int)]
│   └── Основа для будущего экрана истории
│
├── Database (GRDB.swift / SQLite)
│
└── SettingsStore (@AppStorage / UserDefaults)
```

**Поток данных:**
```
[Нажатие клавиши]
  → CGEventTap callback (фоновый поток)
  → atomicAdd(inMemoryCounter)          ← единственное действие в hot path
  → каждые 30 с: flush to SQLite
  → при открытии popover: SQL агрегация → StatsSnapshot → SwiftUI View
```

### 4.4 Требования к производительности

| Параметр | Целевое значение |
|---|---|
| CPU в idle (нет нажатий) | < 0.1% |
| CPU на пике (300 нажатий/мин) | < 0.5% |
| RAM footprint | < 20 МБ |
| Задержка tap-коллбэка | < 100 мкс |
| Запись в SQLite | каждые 30 с, фоновая очередь |
| Время холодного старта | < 300 мс |

## 5. Приватность и безопасность

- **Никакого текста не сохраняется.** В tap-коллбэке не читается ни `kCGKeyboardEventKeycode` (для символа), ни другие поля события — только факт нажатия и временна́я метка.
- `LSUIElement = YES` в `Info.plist` — нет иконки в Dock, нет стандартного меню приложения.
- БД содержит только числа и временны́е метки. Файл доступен только текущему пользователю.
- Требуемое системное разрешение: **Input Monitoring** (`Privacy & Security → Input Monitoring`).

## 6. Дистрибуция и сборка из исходников

### 6.1 Модель распространения

- **Репозиторий GitHub**, лицензия MIT.
- **Releases**: нотаризованный `.dmg`, собирается через GitHub Actions.
- Версия приложения берётся из git tag (`v1.0.0`).

### 6.2 Сборка из исходников

Пользователь клонирует репозиторий, открывает `KeyCount.xcodeproj` в Xcode, меняет `Bundle Identifier` и `Team ID` на свои, собирает и запускает. Инструкция включается в `README.md`.

**Ключевая особенность:** разрешение Input Monitoring привязано к подписи бинарника. При `Clean Build` в Xcode (debug) macOS может запросить разрешение повторно — это нормальное поведение при смене подписи. В `README` это объясняется явно.

### 6.3 Entitlements (без App Sandbox)

```xml
<key>com.apple.security.app-sandbox</key>
<false/>
<key>com.apple.security.input-monitoring</key>
<true/>
```

### 6.4 CI/CD

```
GitHub Actions:
  trigger: push tag v*.*.*
  steps:
    → xcodebuild archive
    → xcodebuild -exportArchive
    → xcrun notarytool submit
    → xcrun stapler staple
    → create-dmg → GitHub Release
```

## 8. Задел под историю (v2+)

В v1 экрана истории нет, но реализуются:

- Поминутные бакеты (позволяют любую агрегацию без миграции).
- Слой `StatsRepository` с полным API для работы с диапазонами дат.
- `allDays()` возвращает список `(Date, Int)` — готова для графика и списка.

Возможные экраны v2:
- Список дней с totals за последние 30 дней.
- Простой линейный график по неделям.
- Экспорт CSV.
